## 3. Выявленные проблемы

### 3.1 Незначительные (Minor)

| # | Проблема | Серьёзность | Описание |
|---|----------|-------------|----------|
| 1 | Валидация регистрации | Low | Сообщение «Введите имя» не всегда отображается в DOM при пустой форме. Возможная причина: проверка происходит в JS, но текст ошибки не рендерится в headless-режиме |
| 2 | Отзывы в сайдбаре мастера | Low | При тесте сразу после логина текст «Отзывы» может не успеть отрендериться (асинхронная загрузка профиля) |
| 3 | Пустая страница чатов для admin | Info | Не баг — у admin просто нет чатов. Customer видит 8 чатов |

### 3.2 Наблюдения

| # | Наблюдение | Описание |
|---|------------|----------|
| 1 | Gateway использует cookie-based auth | Bearer-токены не поддерживаются. Аутентификация через `cookie access_token` или `?token=` параметр. Фронтенд работает через `credentials: 'include'` |
| 2 | WebSocket циклы connect/disconnect | При SPA-навигации WebSocket соединения закрываются и пересоздаются — нормальное поведение |
| 3 | React warning в ChatPage | Error boundary срабатывает при отсутствии чатов |
| 4 | MapLibre GL | Карта корректно инициализируется (GPU warning в headless — ожидаемо) |

## 4. Технические детали

### 4.1 Роутинг (App.tsx)
Всего 15 маршрутов. Корректно работают:
- Публичные: `/login`, `/register`
- Защищённые (ProtectedRoute): `/`, `/orders`, `/orders/create`, `/orders/:id`, `/chats`, `/profile`, `/profile/:userId`, `/notifications`, `/reviews`
- Админские (AdminRoute): `/admin`

## 5. Рекомендации

1. **Добавить поддержку Bearer-токенов** в API Gateway — сейчас только cookie-based auth, что неудобно для прямых API-запросов
2. **Улучшить обработку пустых состояний** на странице чатов — показывать заглушку «У вас пока нет чатов» вместо пустой страницы
3. **Добавить Error Boundary** для ChatPage корректно — сейчас React warning при рендере
4. **WebSocket reconnection** — добавить механизм переподключения при обрыве соединения
